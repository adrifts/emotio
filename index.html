OBJECTIVO

Descobrir se é possível representar-se fidedignamente estados emocionais visualmente e comparar dados reais com a experiência pessoal de cada indivíduo.

CONCEITO

Criar uma narrativa visual com conteúdo já existente usando uma compilação de vídeos pré selecionados e captar a reacções emocionais do espectador de forma a que o seu estado emocional seja representado ao vivo, com transformações de imagem e som no vídeo (narrativa) que está a ser visualizado.

ESPECIFICAÇÃO TÉCNICA

Uma compilação/playlist de vídeos de forma a construir uma narrativa visual. Os dados sao captados por um Arduino com um sensor de batimento cardíaco em simultâneo com a visualização da narrativa. Os dispositivo fica montado no dedo da pessoa e faz uma captação simultânea.

Os dados são depois usados no Processing onde os seus valores influenciam o efeito visual e a sua montagem alterando aspectos como a cor, saturação, contraste e velocidade.

FERRAMENTAS UTILIZADAS

  • Adobe Premiere
  • Processing
  • Arduino
  • Sensor de batimento cardíaco

PROCESSO

Nós começamos por usar uma placa Bitalino desenhada especificamente para captar bio-sinais do corpo humano. Como a placa já é pré-programada tínhamos que conseguir exportar os dados captados de forma ao Processing conseguir ler.

O sotware do Bitalino (Open Signals) só exporta dados em H5, um formato desenvolvido pela NASA para guardar grandes quantidades de informação. Isto apresentou-nos problemas graves pois não conseguimos transformar a informação deste ficheiro em tables de forma a que o Processing as conseguisse decifrar. Devido a isto tentamos contornar o problema utilizando um script python programado para captar a informação vinda do Bitalino.

Esta alternativa acabou por custar-nos bastante tempo e no final tivemos que a descartar também. Nós conseguimos conectar o Python ao Bitalino e receber dados viáveis, no entanto, não fomos capazes de descriptar ou "traduzir" os valores de forma a criar variáveis para serem usadas no Processing em tempo real.

A partir daí, avançamos para a nossa solução final. Nós substituímos a placa Bitalino por um Arduino juntamente com os sensores do Bitalino e utilizamos um simples analog read para recolher dados. Com esta alternativa conseguimos ler os dados com facilidade e usa-los no nosso código de Processing. Contudo, os sensores do Bitalino provaram serem difíceis de usar, muitas vezes resultando em discrepâncias de valores sem um motivo óbvio. Decidimos então usar um sensor de batimento cardíaco mais simples para captar os dados.

Durante todo este processo desenvolvemos também um código Processing para criar uma interface que permite ver graficamente as pulsações cardíacas em tempo real e um segundo gráfico que demonstra o intervalo entre as pulsações. Isto é importante para podermos ver se o sinal está estável. Inicialmente o código vai ler as portas activas e permite o utilizador seleccionar uma e os gráficos e vídeo só começam após essa selecção. Finalmente um indicador de pulsações por minuto. Ao mesmo tempo corre a narrativa visual, alterada pela pulsação, esta pode ser colocada em full-screen ocultando os gráficos.
